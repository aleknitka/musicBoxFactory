# Plan 05-01: User Interface Scaffolding and API

**Goal:** Implement the FastAPI backend and generation logic for the web UI.

## Context
This plan lays the foundation for the web interface by creating the FastAPI application, Pydantic models for request validation, and the logic to interface with the `musicboxfactory` library.

## Tasks
- [x] Add `fastapi`, `gradio`, and `uvicorn` to `pyproject.toml` dependencies.
- [x] Create `src/musicboxfactory/ui/` directory.
- [x] Implement `src/musicboxfactory/ui/models.py` with Pydantic request models.
- [x] Implement `src/musicboxfactory/ui/logic.py` for audio generation and temporary file handling.
- [x] Implement `src/musicboxfactory/ui/app.py` with initial FastAPI endpoints.
- [x] Create `tests/test_ui_api.py` to verify the generation endpoint.
- [x] Ensure `uv run pytest tests/test_ui_api.py` passes.

## Success Criteria (Must-Haves)
1. `POST /api/v1/generate` accepts audio parameters and returns a path to a generated WAV file.
2. Generated WAV file is saved in a temporary directory.
3. The API properly handles missing soundfont errors by returning a 400 Bad Request.
4. FastAPI application is importable and runs via `uvicorn musicboxfactory.ui.app:app`.

## Verification Strategy
- **Manual Verification:** Use `curl` or FastAPI's Swagger UI (`/docs`) to trigger audio generation and check the output path.
- **Automated Testing:** Use `fastapi.testclient.TestClient` in `tests/test_ui_api.py` to mock parameters and verify JSON responses.
- **File System Check:** Confirm that temporary files are actually written and are valid WAVs.
