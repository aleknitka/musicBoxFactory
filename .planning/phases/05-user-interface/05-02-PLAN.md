# Plan 05-02: Gradio UI and Integration

**Goal:** Build the Gradio interface and integrate it into the FastAPI application.

## Context
With the backend API and generation logic in place, this plan focuses on building the user-facing Gradio UI and mounting it as a sub-application within FastAPI.

## Tasks
- [x] Implement `src/musicboxfactory/ui/interface.py` with the Gradio UI definition.
- [x] Update `src/musicboxfactory/ui/app.py` to mount the Gradio app at `/ui`.
- [x] Add instructions and basic styling to the Gradio UI.
- [x] Implement a manual test script `scripts/run_ui.py` to launch the full application for manual verification.
- [x] Update `README.md` with instructions on how to run the UI.

## Success Criteria (Must-Haves)
1. Gradio UI is accessible at `http://localhost:8000/ui`.
2. UI includes all necessary inputs (sf2 path, instrument, melody, ambient, volumes, duration).
3. "Generate" button triggers the API/logic and displays the audio player with the result.
4. The application starts without errors and is stable during basic interaction.

## Verification Strategy
- **Manual Verification:** Run the app using the test script, navigate to `/ui`, and perform end-to-end generation tests.
- **Integration Check:** Verify that the UI correctly handles the case where the SF2 file is missing (shows error message).
- **Static File Serving:** Ensure the `gr.Audio` component can correctly play the files served from `/static`.
